HTTP Methods Testing

GET /get
Test GET requests and view request information
GET:curl "http://localhost:5000/get?param1=value1¶m2=value2"
POST /post
Test POST requests with form data or JSON
POST:curl -X POST "http://localhost:5000/post" -H "Content-Type: application/json" -d '{"key":"value","number":42}'
PUT /put
Test PUT requests with data
PUT:curl -X PUT "http://localhost:5000/put" -H "Content-Type: application/json" -d '{"update":"data"}'
DELETE /delete
Test DELETE requests
DELETE:curl -X DELETE "http://localhost:5000/delete"
PATCH /patch
Test PATCH requests with partial data
PATCH:curl -X PATCH "http://localhost:5000/patch" -H "Content-Type: application/json" -d '{"field":"updated_value"}'
HEAD /head
Test HEAD requests (headers only)
HEAD:curl -I "http://localhost:5000/head"
OPTIONS /options
Test OPTIONS requests
OPTIONS:curl -X OPTIONS "http://localhost:5000/options"

Status Codes

GET POST PUT PATCH OPTIONS /status/<code>
Return response with specific HTTP status code (e.g., /status/404). Supports multiple HTTP methods.
GET:curl -L "http://localhost:5000/status/301"
POST:curl -X POST "http://localhost:5000/status/404"
PUT:curl -X PUT "http://localhost:5000/status/200"
PATCH:curl -X PATCH "http://localhost:5000/status/500"
OPTIONS:curl -X OPTIONS "http://localhost:5000/status/204"
GET /status/random
Return response with random status code. Only supports GET method.
GET:curl "http://localhost:5000/status/random"

Cache

GET /cache
Test HTTP caching. Returns 304 (Not Modified) if If-Modified-Since or If-None-Match headers are present, otherwise returns data with Last-Modified and ETag headers.
GET:curl -i "http://localhost:5000/cache"
Conditional:curl -i -H "If-Modified-Since: Thu, 01 Jan 1970 00:00:00 GMT" "http://localhost:5000/cache"
ETag:curl -i -H "If-None-Match: some-etag" "http://localhost:5000/cache"
GET /cache/<seconds>
Set Cache-Control header for specified number of seconds (e.g., /cache/300 for 5 minutes).
GET:curl -i "http://localhost:5000/cache/300"
1 hour:curl -i "http://localhost:5000/cache/3600"
GET /etag/<etag>
Test ETag handling. Returns 304 if If-None-Match matches, 412 if If-Match doesn't match (and not wildcard), otherwise returns normal response with the specified ETag.
GET:curl "http://localhost:5000/etag/test123"
Match:curl -i -H "If-None-Match: test123" "http://localhost:5000/etag/test123"
Wildcard:curl -i -H "If-None-Match: *" "http://localhost:5000/etag/test123"
If-Match:curl -i -H "If-Match: other-etag" "http://localhost:5000/etag/test123"

Dynamic Data

GET /delay/<seconds>
Return delayed response with timing information. Maximum delay is 60 seconds. Response includes actual delay time and timestamps.
GET:curl "http://localhost:5000/delay/3"
10 sec:curl "http://localhost:5000/delay/10"
Timing:time curl "http://localhost:5000/delay/5"
Max:curl "http://localhost:5000/delay/60"
Error:curl "http://localhost:5000/delay/120"
GET /base64/encoding/<value>
Encode string to base64url format. Useful for testing base64 encoding/decoding in APIs.
Encode:curl "http://localhost:5000/base64/encoding/hello"
Text:curl "http://localhost:5000/base64/encoding/Hello%20World"
JSON:curl "http://localhost:5000/base64/encoding/$(echo -n '{"key":"value"}' | base64 -w 0)"
GET /base64/decoding/<value>
Decode base64url-encoded string. Returns decoded text or error message for invalid input.
Decode:curl "http://localhost:5000/base64/decoding/aGVsbG8="
Example:curl "http://localhost:5000/base64/decoding/aGVsbG93b3JsZA=="
Invalid:curl "http://localhost:5000/base64/decoding/invalid-base64"
GET /bytes/<n>
Generate n random bytes (max 1MB). Supports optional 'seed' parameter for reproducible output. Returns binary data with application/octet-stream content type.
100 bytes:curl "http://localhost:5000/bytes/100" | wc -c
With seed:curl "http://localhost:5000/bytes/50?seed=12345" | hexdump -C
1KB:curl "http://localhost:5000/bytes/1024" | wc -c
Save file:curl "http://localhost:5000/bytes/500" -o random.bin
GET /uuid
Generate a random UUID4. Returns JSON with UUID and timestamp. Useful for testing unique identifier generation.
GET:curl "http://localhost:5000/uuid"
JSON:curl -s "http://localhost:5000/uuid" | python -m json.tool
Multiple:for i in {1..3}; do curl -s "http://localhost:5000/uuid" | jq .uuid; done
GET /stream/<n>
Stream n JSON responses (max 100). Each response includes request info with incremental ID. Useful for testing streaming data processing.
5 items:curl "http://localhost:5000/stream/5"
Line count:curl -s "http://localhost:5000/stream/10" | wc -l
Process:curl -s "http://localhost:5000/stream/3" | while read line; do echo "Got: $line"; done
GET /stream-bytes/<n>
Stream n random bytes (max 100KB) with configurable chunk size. Supports 'seed' for reproducible output and 'chunk_size' parameter (default 10KB).
1KB:curl "http://localhost:5000/stream-bytes/1024" | wc -c
With seed:curl "http://localhost:5000/stream-bytes/500?seed=42" | hexdump -C | head
Small chunks:curl "http://localhost:5000/stream-bytes/100?chunk_size=10" | wc -c
Save stream:curl "http://localhost:5000/stream-bytes/2048?seed=123" -o stream.bin
GET /drip
Drip data over a specified duration with optional initial delay. Supports parameters: duration (default 2s), numbytes (default 10, max 10MB), code (status code, default 200), delay (initial delay, default 0).
Basic:curl "http://localhost:5000/drip"
Custom:curl "http://localhost:5000/drip?duration=5&numbytes=50"
With delay:curl "http://localhost:5000/drip?delay=2&duration=3&numbytes=20"
Timing:time curl "http://localhost:5000/drip?duration=4&numbytes=100"
GET /links/<n>/<offset>
Generate HTML page with n links to other pages (1-200 links). Useful for testing web crawlers and link following. The offset parameter highlights the current page.
10 links:curl "http://localhost:5000/links/10/0"
Different page:curl "http://localhost:5000/links/10/5"
Max links:curl "http://localhost:5000/links/200/0" | wc -l
Extract links:curl -s "http://localhost:5000/links/5/2" | grep -o 'href="[^"]*"'
GET /range/<numbytes>
Support HTTP range requests for partial content (max 100KB). Supports Range header for byte ranges, plus chunk_size and duration parameters. Returns 206 for partial content, 416 for invalid ranges.
Full:curl -i "http://localhost:5000/range/100"
Range:curl -H "Range: bytes=0-9" -i "http://localhost:5000/range/100"
Last 10:curl -H "Range: bytes=-10" -i "http://localhost:5000/range/100"
From 50:curl -H "Range: bytes=50-" -i "http://localhost:5000/range/100"
Invalid:curl -H "Range: bytes=200-300" -i "http://localhost:5000/range/100"

Cookie Management

GET /cookies
Return cookies sent by client

Add random cookies to response for testing
GET:curl -c cookies.txt "http://localhost:5000/cookies/add"

Redirects

GET /redirect/<n>
Perform n redirects (302) with optional absolute/relative control. Use ?absolute=true for absolute redirects.
3 redirects:curl -L "http://localhost:5000/redirect/3"
Absolute:curl -L "http://localhost:5000/redirect/2?absolute=true"
No follow:curl -i "http://localhost:5000/redirect/1"
GET /absolute-redirect/<n>
Perform n absolute redirects (302). Always uses absolute URLs in Location header.
2 redirects:curl -L "http://localhost:5000/absolute-redirect/2"
Headers:curl -i "http://localhost:5000/absolute-redirect/1"
GET /relative-redirect/<n>
Perform n relative redirects (302). Uses relative URLs in Location header.
3 redirects:curl -L "http://localhost:5000/relative-redirect/3"
Headers:curl -i "http://localhost:5000/relative-redirect/1"
Verbose:curl -v -L "http://localhost:5000/relative-redirect/2"
